# User Stories

## Overview

The Privacy Dashboard is developed using Scrum. Development is organized around user stories that describe the needs of different stakeholders, such as ML engineers, data protection officers, project leads, administrators, and general users.

Each user story follows the format:

> **As a** <role>, **I want** <feature>, **so that** <benefit>.

## Must-have User Stories

| ID | User Story |
|----|------------|
| **US-01** | As an ML Engineer, I want to create a new project with a name and description so that I can organize my work. |
| **US-02** | As an ML Engineer, I want to add data sources to every project. |
| **US-03** | As an ML Engineer, I want to see whether a dataset contains personal data, including GDPR Art. 9 data. |
| **US-04** | As a user, I want to see a traffic-light overview of all projects so that I can quickly identify projects requiring attention. |
| **US-05** | As a user, I want to log in so that only authorized users can access project data. |
| **US-06** | As an ML Engineer, I want to view a project's details, including data sources, metrics, and risk assessment. |
| **US-07** | As a user, I want the dashboard to support different types of data sources, such as text and images. |

## Should-have User Stories

| ID | User Story |
|----|------------|
| **US-08** | As a Data Protection Officer, I want to export a privacy report as a PDF. |
| **US-09** | As an ML Engineer, I want to import data sources and metrics from a JSON file. |
| **US-10** | As a Project Lead, I want to create different versions of data sources to track privacy status over time. |
| **US-11** | As a Data Protection Officer, I want to receive recommendations when a project has a yellow or red risk level. |
| **US-12** | As a user, I want to preview datasets directly in the dashboard. |

## Nice-to-have User Stories

| ID | User Story |
|----|------------|
| **US-13** | As an administrator, I want to manage users with different roles and permissions. |
| **US-14** | As an ML Engineer, I want to track changes to data sources and metrics in a version history. |
| **US-15** | As a user, I want to use the dashboard in dark mode. |
